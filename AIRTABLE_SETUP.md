# Як підключити Airtable до сайту

Два кроки: **(1) налаштувати таблицю** + **(2) підключити сайт через env vars**.

---

## КРОК 1 — Створити таблицю «Товари» в Airtable

### 1.1. Відкрий Airtable

Зайди на https://airtable.com → у тебе вже є база `appSUzIDbLksvmdkq` (та сама де бот). Створи там нову таблицю **«Товари»** (Add a table).

### 1.2. Створи поля точно з такими назвами і типами

> 🔥 **Назви полів важливі — латиницею як нижче, інакше сайт не знайде**

| Назва поля | Тип Airtable | Примітка |
|------------|--------------|----------|
| `id` | Single line text | Slug для URL: `micotin-monarda`, `wartoff-1` |
| `name` | Single line text | Назва українською |
| `brand` | Single Select | Опції: `Veratin`, `DivaPharm` |
| `category` | Single Select | Опції: див. нижче |
| `tags` | Multiple Select | Опції: див. нижче |
| `variants_display` | Single line text | «50 мл — 530 грн \| 200 мл — 2000 грн» |
| `supplier_min_price` | Number | Ціна постачальника (мін) |
| `supplier_max_price` | Number | Ціна постачальника (макс) |
| `currency` | Single Select | Опції: `USD`, `UAH` |
| **`price_uah`** | **Currency** (UAH) | **🟡 Твоя роздрібна ціна для клієнта** |
| `short` | Long text | Короткий опис для картки |
| `long` | Long text | Повний опис для сторінки товару |
| `usage` | Long text | Спосіб застосування |
| `inci` | Long text | Склад |
| `image` | URL | Посилання на фото (поки що зовнішнє) |
| `source` | URL | Посилання на сторінку постачальника |
| `status` | Single Select | Опції: `ready`, `todo` |
| **`in_stock`** | **Checkbox** | **🟡 Чи є на сайті — ти ставиш галку** |
| **`featured`** | **Checkbox** | **🟡 Показувати на головній як хіт** |

**Опції для Single/Multi Select (створюються по ходу):**

`category`: `Догляд за нігтями`, `Догляд за шкірою рук`, `Догляд за шкірою ніг`, `Від бородавок та папілом`, `Протигрибкові засоби`, `Універсальні засоби`, `Професійні засоби`

`tags`: `Креми для рук`, `Для проблемних нігтів`, `Гіпергідроз`, `Дезодоранти`, `Кератолітики`, `Аксесуари`, `Інструменти`, `Для дітей`, `Антибактеріальні засоби`, `Для проблемної шкіри`, `Від бородавок` *(може ще додатися — Airtable дозволяє створювати нові опції при імпорті)*

### 1.3. Імпортуй CSV

1. У таблиці «Товари» натисни **+** поряд з останнім стовпцем → **Import data** → **CSV file**
2. Обери файл `master_catalog.csv` з попереднього архіву
3. Airtable запитає мапу полів — повинні всі співпасти за назвами автоматично
4. Натисни **Import**

✅ Після імпорту в тебе буде 39 товарів. Поля `price_uah`, `in_stock`, `featured` будуть порожні — їх заповнюєш ти вручну.

### 1.4. Заповни що треба заповнити

Для кожного товару:
- **`price_uah`** — твоя роздрібна ціна в гривнях
- **`in_stock`** — ✅ галка для тих, що готові продавати на сайті (хто без галки — не показується)
- **`featured`** — ✅ галка для тих, що показувати на головній як хіти (4-6 штук)

## КРОК 2 — Створити Airtable Personal Access Token

Це ключ, через який сайт читає базу.

1. Відкрий https://airtable.com/create/tokens
2. **Create token** → дай йому імʼя: `Beauty & Shine site`
3. **Scopes**: додай тільки `data.records:read` (більше нічого не треба — лише читання)
4. **Access**: обери базу `Beauty & Shine` (та сама де таблиця «Товари»)
5. **Create token** → скопіюй значення, воно починається з `pat...` — потрібно зберегти, бо потім не побачиш

⚠️ Цей токен — як пароль. Нікому не показуй.

---

## КРОК 3 — Підключити сайт через Vercel env vars

1. Відкрий свій проект на https://vercel.com
2. **Settings** → **Environment Variables**
3. Додай **два значення**:

| Name | Value | Environments |
|------|-------|--------------|
| `AIRTABLE_TOKEN` | твій токен (`pat...`) | Production, Preview, Development |
| `AIRTABLE_BASE_ID` | `appSUzIDbLksvmdkq` | Production, Preview, Development |

4. Натисни **Save**
5. **Deployments** → правий клік на останньому деплої → **Redeploy** (треба, щоб env vars підхопились)

Через ~1 хвилину сайт оновиться і **підтягне товари з Airtable**. Перевір:
- `/catalog` — повинен показати всі товари з `in_stock=true`
- `/product/micotin-monarda` — окрема сторінка товару
- Головна — у блоці «Хіти» товари з `featured=true`

---

## Як працює оновлення

Сайт **кешує** дані з Airtable на 60 секунд. Якщо змінив товар у таблиці — за хвилину сайт сам підтягне нові дані. Перевантажувати не треба.

Якщо щось не зʼявляється:
- Перевір що `in_stock` стоїть галка
- Перевір що `name` і `id` заповнені
- Через 60 секунд оновиться

---

## Що далі за нашим планом

Це **Етап 2 завершено** — каталог підключений до Airtable, товари відображаються на сайті.

Етап 3 буде **кошик і оформлення**: клієнт додає товар у кошик, оформлює замовлення з адресою і телефоном, замовлення зберігається назад в Airtable (нова таблиця «Замовлення») і приходить алертом тобі в бота.

Скажи, коли заімпортуєш каталог і виставиш в Airtable галки — тоді переходимо до Етапу 3.
